Privacy Policy

Effective Date: March 13, 2026  ·  Last Updated: March 13, 2026

Short version: InfoLens reads device information locally on your device only. We do not collect, store, transmit, or share any personal data with any server or third party. Ever.

1 Introduction

InfoLens ("we", "our", or "the app") is a device information utility for Android that displays hardware and software details about your phone directly on screen. This Privacy Policy describes what device information the app reads, why it reads it, and confirms that no data leaves your device.

2 Information We Access

InfoLens reads the following information locally on your device solely to display it within the app. Nothing is uploaded, logged, or shared.

Device & Hardware

  • Device model, manufacturer, brand, and hardware identifiers
  • Android version, build number, security patch level, kernel version, and bootloader version
  • CPU architecture, core count, and real-time core frequencies
  • RAM usage, storage capacity, and free storage
  • Display resolution, refresh rate, and pixel density
  • Battery level, charging status, temperature, and health

Network

  • Wi-Fi connection status, signal strength (RSSI), frequency band, link speed, and Wi-Fi standard
  • Wi-Fi SSID and BSSID requires Location permission
  • Local IP address, gateway, subnet mask, and DNS addresses
  • Mobile network type and carrier name

System & Software

  • Installed application count
  • Sensor count and sensor types
  • Bluetooth support status
  • SIM and eSIM support status

Camera & Microphone Tests only

  • Camera hardware presence and capabilities — read-only, no images captured or stored
  • Microphone hardware presence — used only for the hardware test, no audio recorded or stored

3 How We Use This Information

All accessed information is:

  • Displayed on-screen to you in real time
  • Never stored to any file, database, or remote server
  • Never transmitted off your device
  • Never shared with any third party, advertiser, or analytics service

4 Permissions Explained

PermissionWhy It Is Needed
ACCESS_FINE_LOCATION Required by Android to read the Wi-Fi network name (SSID). No location data is stored or transmitted.
ACCESS_COARSE_LOCATION Fallback location access for Wi-Fi network information on older Android versions.
ACCESS_WIFI_STATE Read Wi-Fi connection details (signal strength, IP address, frequency, etc.).
ACCESS_NETWORK_STATE Detect network connectivity type (Wi-Fi, mobile data).
CHANGE_NETWORK_STATE Used to query active network properties.
READ_PHONE_STATE Read carrier name and SIM information displayed in the Device tab.
BLUETOOTH / BLUETOOTH_CONNECT Detect Bluetooth hardware support and version.
RECORD_AUDIO Used only during the microphone hardware test. No audio is ever recorded, saved, or transmitted.
VIBRATE Used during the vibration hardware test in the Tests tab.
INTERNET Required by the React Native framework. InfoLens makes no outbound network requests.

You may deny optional permissions (Location, Microphone). The app will continue to function; only the specific features requiring those permissions will show a prompt.

5 Data Storage

InfoLens stores no user data of any kind:

  • No databases or local files
  • No analytics or crash reporting SDKs
  • No advertising SDKs
  • No cloud sync or backup

6 Third-Party Services

InfoLens does not integrate any third-party SDKs for analytics, advertising, crash reporting, or social platforms. The app uses only standard Android APIs and open-source React Native libraries for UI rendering.

7 Children's Privacy

InfoLens does not knowingly collect any information from children under the age of 13 (or the applicable age in your jurisdiction). Since no data is collected at all, the app is safe for all ages.
